# Поисковая система Search Engine v1.0

Дипломный проект. Поисковая система для текстовых документов в файловой системе с индексацией и ранжированием.

## Структура проекта

SearchEngine/
├── CMakeLists.txt              # Система сборки CMake
├── config.ini                  # Файл конфигурации
├── README.md                   # Документация
├── src/
│   ├── config_reader.h/.cpp    # Парсинг INI-конфигурации
│   ├── db_manager.h/.cpp       # Подключение к БД, создание таблиц
│   ├── file_scanner.h/.cpp     # Рекурсивный обход файловой системы
│   ├── text_processor.h/.cpp   # Токенизация и подсчёт частот
│   ├── thread_pool.h/.cpp      # Пул потоков для многопоточной обработки
│   ├── indexer_main.cpp        # Главный файл индексатора
│   └── searcher_main.cpp       # Главный файл поисковика

## Архитектура
Система состоит из двух независимых программ:

### Индексатор (indexer)
Запускается для предварительного анализа текстовых файлов и построения поискового индекса.

	Алгоритм работы:
1. Чтение конфигурации из `config.ini`.
2. Подключение к PostgreSQL, создание таблиц (если их нет).
3. Рекурсивный обход указанной директории, фильтрация по расширениям.
4. Многопоточное чтение файлов и токенизация (без обращений к БД).
5. Однопоточная запись результатов в БД (исключает deadlock'и).

	Токенизация:
- Удаление знаков препинания, перевод в нижний регистр.
- Фильтрация слов короче 3 и длиннее 32 символов.
- Подсчёт частоты каждого слова в документе.

### Поисковик (searcher)
Консольная программа для выполнения поисковых запросов.

	Алгоритм работы:
1. Чтение конфигурации, подключение к БД.
2. Цикл ввода запросов (до 4 слов через пробел).
3. SQL-запрос: поиск документов, содержащих ВСЕ слова запроса.
4. Ранжирование по сумме частот слов в документе.
5. Вывод до 10 результатов, отсортированных по релевантности.

## Схема базы данных

### Таблица documents
| Поле      |  Тип                 | Описание                 |
|-----------|----------------------|--------------------------|
| id        | SERIAL PK            | Уникальный идентификатор |
|-----------|----------------------|--------------------------|
| file_path | TEXT UNIQUE NOT NULL | Полный путь к файлу      |
|-----------|----------------------|--------------------------|
| file_name | TEXT NOT NULL        | Имя файла                |
|-----------|----------------------|--------------------------|
| indexed_at| TIMESTAMP            | Время индексации         |

### Таблица words
| Поле | Тип                         | Описание                 |
|------|-----------------------------|--------------------------|
| id   | SERIAL PK                   | Уникальный идентификатор |
|------|-----------------------------|--------------------------|
| word | VARCHAR(32) UNIQUE NOT NULL | Слово                    |

### Таблица word_frequencies
| Поле        | Тип                          | Описание                  |
|-------------|------------------------------|---------------------------|
| document_id | INT REFERENCES documents(id) | Ссылка на документ        |
|-------------|------------------------------|---------------------------|
| word_id     | INT REFERENCES words(id)     | Ссылка на слово           |
|-------------|------------------------------|---------------------------|
| frequency   | INT NOT NULL                 | Частота слова в документе |

Связь «многие-ко-многим» между документами и словами реализована через таблицу word_frequencies

## SQL-запрос поиска

SELECT d.file_name, d.file_path, SUM(wf.frequency) AS relevance
FROM word_frequencies wf
JOIN documents d ON wf.document_id = d.id
JOIN words w ON wf.word_id = w.id
WHERE w.word = ANY($1)
GROUP BY d.id, d.file_name, d.file_path
HAVING COUNT(DISTINCT w.word) = $2
ORDER BY relevance DESC
LIMIT $3;


Ключевое условие: HAVING COUNT(DISTINCT w.word) = N — гарантирует, что документ содержит все N слов запроса.

## Многопоточность

Индексатор использует двухфазную обработку для исключения состояний гонки и deadlock'ов:

Фаза 1 (многопоточная): Пул потоков читает файлы и выполняет токенизацию. Каждый поток работает только со своим файлом, результаты сохраняются в общий вектор под защитой мьютекса.
Фаза 2 (однопоточная): Главный поток последовательно записывает результаты в БД. Нет конкурентных блокировок строк

## Конфигурация (config.ini)

[database]
host=localhost          # Адрес сервера PostgreSQL
port=5432               # Порт
dbname=search_engine    # Имя базы данных
user=postgres           # Пользователь
password=secret         # Пароль

[indexer]
start_dir=C:/путь    # Директория для индексации
extensions=txt,md,ini,csv      # Расширения файлов
thread_pool_size=4             # Количество потоков

[searcher]
max_results=10          # Максимум результатов поиска
max_query_words=4       # Максимум слов в запросе

## Зависимости

- Язык: C++17/20
- Сборка: CMake 3.16+
- Компилятор: MSVC 2022 (Visual Studio)
- База данных: PostgreSQL 16
- Библиотеки:
  - Boost (locale, system)
  - libpqxx (C++ клиент PostgreSQL)
  - libpq (C клиент PostgreSQL)
- Пакетный менеджер: vcpkg

## Сборка и запуск
### 1. Установка зависимостей

cd C:\vcpkg
.\vcpkg install boost-locale boost-system libpqxx libpq --triplet x64-windows
.\vcpkg integrate install

### 2. Настройка и сборка

#### Через Visual Studio (рекомендуется)

1. Открыть папку проекта: Файл → Открыть → Локальная папка.
2. Visual Studio автоматически настроит CMake.
3. В выпадающем списке конфигураций выбрать x64-Release.
4. **Сборка → Собрать всё (Ctrl+Shift+B).

> Если библиотеки не найдены, указать путь к vcpkg: Проект → Настройки CMake → cmakeCommandArgs, добавить -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake

#### Из командной строки (альтернативный способ)

mkdir build && cd build
cmake ..
cmake --build . --config Release

### 4. Подготовка базы данных

cd "C:\путь\PostgreSQL\16\bin"
.\initdb.exe -D C:\PostgreSQL_data -E UTF8 --locale=C
.\pg_ctl.exe start -D C:\PostgreSQL_data
.\createuser.exe -s postgres
.\createdb.exe -U postgres -E UTF8 -T template0 search_engine

### 5. Запуск индексатора

.\indexer.exe config.ini

### 6. Запуск поисковика

.\searcher.exe config.ini

Ввести запрос (до 4 слов через пробел) и нажать Enter. Команды:
- :help — справка
- :exit — выход
